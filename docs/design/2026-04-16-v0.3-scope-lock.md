## Why this doc exists

Earlier in the 2026-04-16 session the decision was "no v0.4, everything in v0.3." That was directionally right вЂ” one loud launch, not a trickle вЂ” but operationally it expanded the v0.3 backlog to 35 items, which is over-scoped for a solo founder and dilutes the launch narrative.

Grok's CoS review pushed back: **ring-fence v0.3 around a single coherent story, push the rest to post-launch monthly plugin releases.** That reframe is correct: the plugin architecture means post-launch plugins are their own news cycles, not footnotes. "v0.3 is the foundation; new engines and sinks land monthly" is a momentum story, not a dilution.

## The v0.3 launch narrative (one sentence)

> **AgentKMS catches credential leaks before they cost you. When GitHub secret scanning reports a leaked PAT, your security engineer runs one command and gets a complete forensic report вЂ” and often learns the credential expired hours before the leak was even reported.**

Every v0.3 item must serve that sentence. Items that don't, ship after launch.

## Messaging adjustments per Grok feedback

- **Dual framing for plugins.** DevOps / SRE audience sees "plugins (like Terraform / Vault)." CISO / security-buyer audience sees "Catalyst9 Enterprise Pack with managed security boundaries." Same binary, different emphasis on different surfaces (developer docs vs. enterprise sales pages).
- **Demo timing discipline.** Every launch-content piece (blog, video, README) benchmarks against the <30s forensics report render. If it takes longer to explain the demo than to run it, rewrite.
- **"GitHub first" applies to narrative, not engineering.** `dynsecrets-aws` is still a P0 v0.3 item вЂ” but demos, blog posts, and README walkthroughs lead with GitHub. AWS shows up as the "yes, it does the heavy infra use case too" beat.

## Open questions for the next review cycle

- **License-file durability long-term.** Grok flagged: enterprise buyers with thousands of seats could pirate. Mitigations (machine fingerprints, opt-in telemetry in exchange for value-add, enterprise SLA bundling) worth exploring for post-launch.
- **Wasm plugin runtime.** Grok validated `hashicorp/go-plugin` for v0.3. Wasm as a parallel plugin runtime for future-proofing is a v0.4/0.5 experiment вЂ” prototype in a branch, don't block on it.
- **Narrow-hosted Dynamic Secrets tier pricing.** The hosted-operator tier (Catalyst9 runs AgentKMS in customer's VPC) is already in the deployment model. Pricing model and minimum-contract sizing needs work before we solicit pilot customers вЂ” but that's a post-launch problem.
